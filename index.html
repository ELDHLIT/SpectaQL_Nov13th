Subscriptions

endpoint

Description

Comment N°290 VOS Ad Controller (VAC) API supports the ability to subscribe to changes in the API.

This is done through Webhook Subscription using Azure Event Grid.

A webhook is an HTTP-based callback function that allows lightweight, event-driven communication between 2 APIs.

To set up a webhook, the client gives a unique URL to the server API and specifies which event it wants to know about

Once the webhook is set up, the client no longer needs to poll the server.

The server will automatically send the relevant payload to the client’s webhook URL when the specified event occurs

Webhook Subscriptions are created via a Private API Endpoint. An end customer cannot therefore create the subscription but it can query an existing subscription.

This allows applications and services that integrate with the VAC API to be notified of updates, additions, or modifications to the API.

Possible API changes: create / update / delete on  account / system / cluster / service / global event settings / live event

Such a subscription is identified by :

  • An Endpoint
  • A Name
  • A Filter
  • A Provisioning State

The Endpoint has two fields :

URL: URL of the webhook callback function basicAuth : Authentication key of the webhook callback function

Response

Returns a SubscriptionEndpoint!

Example

Query
subscription Endpoint {
  endpoint {
    url
    basicAuth
  }
}
Response

SubscriptionEndpoint

Description

Comment N°294
VOS Ad Controller (VAC) API supports the ability to subscribe to changes in the API. This is done through Webhook Subscription using Azure Event Grid. A webhook is an HTTP-based callback function that allows lightweight, event-driven communication between 2 APIs. To set up a webhook, the client gives a unique URL to the server API and specifies which event it wants to know about Once the webhook is set up, the client no longer needs to poll the server. The server will automatically send the relevant payload to the client’s webhook URL when the specified event occurs Webhook Subscriptions are created via a Private API Endpoint. An end customer cannot therefore create the subscription but it can query an existing subscription.

This allows applications and services that integrate with the VAC API to be notified of updates, additions, or modifications to the API.

Possible API changes: create / update / delete on  account / system / cluster / service / global event settings / live event

Such a subscription is identified by :

  • An Endpoint
  • A Name
  • A Filter
  • A Provisioning State

The Endpoint has two fields :

URL: URL of the webhook callback function basicAuth : Authentication key of the webhook callback function

Fields
Field Name Description
url - String! Comment N°295 URL of the webhook callback function
basicAuth - String Comment N°296 Authentication key of the webhook callback function
Example
{
  "url": "xyz789",
  "basicAuth": "abc123"
